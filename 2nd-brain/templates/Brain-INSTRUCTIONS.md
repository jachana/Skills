# Brain protocol for `{{PROJECT}}`

This file is copied into each repo's `.brain/INSTRUCTIONS.md` by the sync script. It's the short, in-repo reminder of the contract. The full skill is `brain-sync`.

You are an AI agent working in this repo. The `.brain/` folder bridges this repo to a central Obsidian vault (the "brain").

## Read on session start
- `.brain/CONTEXT.md` — scope, phase, milestones, team (READ-ONLY)
- `.brain/TASKS.md` — tasks delegated to you (tick `[x]` when done)
- `.brain/ASKS.md` — questions to answer (READ-ONLY)

## Write as you work
- `STATUS.md` — keep frontmatter current (`last-activity`, `progress`, `phase`, `blockers`, `next-action`)
- `LOG.md` — append `- YYYY-MM-DD HH:MM — [TYPE] description` (WORK · DECISION · MILESTONE · BLOCKER · LESSON · HANDOFF)
- `ANSWERS.md` — append `Q:` / `A:` pairs answering items from `ASKS.md`

## Never
- Edit `CONTEXT.md`, `TASKS.md` (beyond ticking), `ASKS.md`, `INSTRUCTIONS.md`.
- Reach into the vault directly — talk only through these files.
- Create files in `.brain/` beyond the 7 standard ones.
