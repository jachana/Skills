# reporting — human-readable output

Skills that turn raw material (work done, data, notes, your second-brain) into something a **person** actually wants to read and act on.

| Skill | Use it when… |
|---|---|
| [`human-readable-report`](human-readable-report/) | You need a status update, summary, or analysis written for a human — leading with the answer, structured, skimmable, no wall of text. |
| [`proposal-writer`](proposal-writer/) | You're writing a proposal, quote, or pitch — scope, deliverables, timeline, price, and a clear call to action. |

Both are **bilingual** (EN/ES) and lean on a "lead with the conclusion, show evidence, cut filler" style — they pair especially well with the **caveman** companion skill (kills hedging) and **frontend-design** when the output becomes a web page.

No env vars, no scripts — pure writing skills.
